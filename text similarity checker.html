Longest Common Subsequence
Dynamic programming matrix — green cells trace the optimal backtrack path

Algorithm Steps

1
Tokenize
Split both texts into tokens (characters or words). Creates sequences A[0..m-1] and B[0..n-1].
2
Build DP Table
Initialize an (m+1)×(n+1) matrix filled with zeros. Each cell dp[i][j] stores the LCS length for the first i and j tokens.
3
Fill Recurrence
If A[i-1] == B[j-1]: dp[i][j] = dp[i-1][j-1]+1
Else: dp[i][j] = max(dp[i-1][j], dp[i][j-1])
4
Backtrack
Trace from dp[m][n] back to dp[0][0] following match decisions to recover the actual subsequence tokens.
5
Compute Similarity
Dice coefficient: 2 × |LCS| / (|A| + |B|) × 100%. Symmetric and length-normalized.

Complexity Analysis

T
Time Complexity
Every cell of the m×n table is computed once → O(m × n)
S
Space Complexity
The full table is stored for backtracking → O(m × n)
Reducible to O(min(m,n)) if only the LCS length is needed (two-row rolling array).
J
Justification
LCS is ideal for plagiarism detection because it tolerates insertions, deletions, and minor rephrasing while detecting preserved sequential structure — unlike exact substring matching or simple word-overlap methods.
